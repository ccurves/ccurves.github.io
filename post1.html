WakaDupe (Tourist Web application)

Recently I just found out that there is a river behind my home and this river actually leads to a hidden village. Have lived here for quite some time this came as a shocker, I made up mind to find this place and when I did it was mind blowing. Describing this to my parents and asking if they were aware of this place but found out it was the first time they were hearing about it also.

It then occurred to me that there could be other places like that around the country, unknown and waiting to be discovered. Hidden attractions that tourist and even locals can enjoy, so I decided to make a web platform to try to solve this issue.

So I created WakaDupe, an app were users can post and comment on attractions from around the country.

WakaDupe is built on a MongoDB/Express/Node stack, utilizing RESTful architecture with the Bootstrap 4 CSS framework for styling.

The app performs CRUD operations for users, the attractions and the comments. These pieces are referenced within the database through various associations. Actionable commands are displayed dynamically on the site (edit/delete) depending on a user’s authorization/ownership. Flash messages handle error and success messages to provide the user with feedback.

I made use of MongoDB Atlas, a cloud-based NoSQL database for data storage. And utilizing Passport js ad related sub-modules I integrated user authorization and authentication for various routes. I added Goggle map feature using Node-Geocoder to enable user see accurate locations of this attractions. The app is deployed using Heroku.

Much of the code has been modularized into separate pieces, including routes, middleware (session authentication), and database models.

I shared the app with friends for debugging and testing and have gotten a lot feedback and additional features that should be added. So future version will extend functionality.
